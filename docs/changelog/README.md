# Changelog

This directory contains changelog entries for tracking project history and releases.

## Structure

- Each changelog entry should be a markdown file with YAML frontmatter
- Files should be named with date prefix: `YYYY-MM-DD-description.md`
- Use the changelog template for consistent formatting

## Template

See `docs/changelog/templates/changelog-entry.md` for the standard template.

## Purpose

Changelogs document:
- New features and enhancements
- Bug fixes
- Breaking changes  
- Deprecations
- Security updates

## Process

1. Create a new changelog entry for each release
2. Update the root CHANGELOG.md with a summary
3. Include version numbers and dates
4. Link to relevant issues/PRs
