# Request for Comments (RFCs)

Technical proposals for architectural changes and major features.

## Purpose

RFCs document:
- Architectural decisions needing review
- Major feature implementations
- Breaking changes
- Integration patterns

## Structure

Follow the RFC template in `docs/rfc/templates/`:
1. Summary and motivation
2. Design details
3. Alternatives considered
4. Implementation plan
5. Open questions

## Process

1. Create RFC draft
2. Circulate for feedback (minimum 2 reviewers)
3. Incorporate feedback
4. Finalize and archive
5. Create implementation tickets from RFC tasks

## Review Criteria

- Technical feasibility
- Consistency with existing architecture
- Backward compatibility considerations
- Security implications
- Performance impact

## Templates

- Standard RFC: `rfc.md`
- Integration RFC: `integration-rfc.md` (for external system integrations)
