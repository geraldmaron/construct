Enhance `construct init-docs` to suggest and optionally organize existing markdown files into appropriate documentation lanes.

1. Add `--suggest-org` and `--organize` flags to init-docs.
2. Implement scanning of markdown files (excluding ignored directories and already in docs/).

## Decisions

- We decided to only suggest files at the repo root or in non-docs top-level dirs (to avoid suggesting files already in docs lanes).
- We decided to require `--yes` with `--organize` to avoid accidental moves.
- We decided to pluralize lane names appropriately (e.g., 'memo' -> 'memos').
